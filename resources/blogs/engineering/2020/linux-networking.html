Suppose you have two services, user-service (which handles authentication) and your feature-service (which provides the feature to your clients). As soon as you receive the request from the user, as first step you would always want to authenticate the user request. So here is how the end-to-end flow looks like,
  1. You receive the user request with all the neccesarry data
  2. The feature-service creates the request object as per the API contract with user-srvice
  3. feature-service passes all the HTTP request data (type, url, data) to the HTTP lib being used
  1. the lib creates a socket to initiate the connection
  2. the lib records this socket-id and sends the (ip, port, socket-id, ...) to establish a TCP connection
  3. the user-service makes accepts this request, creates a socket at its end and makes a note of the incoming socket-id (generally servers implement resource pooling and an idle socket is used) to this assigned socket
  4. the user-service returns its socket-id in the ACK response and the connection is established
  5. the lib copies the data from service buffer to the socket send buffer
  6. the kernel moves this data to one of the avaialble TX ring buffers
  7. the kernel writes the address pointer of this ring buffer into the DMA controller
  8. the DMA controller now reads this data from TX ring buffer and transmits to the NIC
  9. NIC sends this data over wire to the outer world
  1. NIC receives the data
  2. NIC writes the incoming data to the RX ring buffer using the DMA controller and raises an interrupt to the CPU
  3. the kernel copies this data from the ring buffer to the linked socket receive buffer
  4. the kernel then moves this data from the socket receive buffer to the service buffer
  5. the user-service has a pool of threads (and linked sockets) waiting for data from the connection and kernel passes the control to it
  6. the user-service reads this data from the service buffer and completes the data receiving part
  7. user-service can now query the db/cache to authenticate the request
